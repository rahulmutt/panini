# mise Dependency + CI Refresh Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Bump the mise-pinned Rust toolchain to latest stable, pin the cargo dev-tools under mise, refresh the lockfile, migrate to edition 2024, and update CI and docs to match.

**Architecture:** A two-tier mise layout — base `mise.toml` carries only the Rust toolchain (with clippy/rustfmt components) for the fast build/test/lint path; a new `mise.dev.toml` (config-environment `dev`) carries the heavier cargo dev-tools, pulled on demand with `MISE_ENV=dev`. CI drops the manual rustup-clippy workaround and gains a non-blocking supply-chain audit job. Edition migration is mechanical (`cargo fix --edition`) and gated by the existing golden paradigm test.

**Tech Stack:** Rust (cargo workspace, resolver 2), mise (tool pinning + task runner), GitHub Actions, cargo-deny / cargo-audit / cargo-mutants / cargo-fuzz via the mise `cargo:` backend.

## Global Constraints

- Pinned Rust toolchain: **1.97.1** (exact). Never a fuzzy `rust = "1"`.
- Pinned dev-tools (exact): cargo-deny **0.20.2**, cargo-audit **0.22.2**, cargo-mutants **27.1.0**, cargo-fuzz **0.13.2**.
- MSRV (`rust-version`): **1.97**. Edition: **2024**.
- All pins fully resolved (mise `--pin` semantics); commit `Cargo.lock`.
- SLP1 is the only internal representation; the golden paradigm test (`crates/panini/tests/paradigm.rs`) is the grammar acceptance gate — surface forms must not change.
- `#![forbid(unsafe_code)]` holds in every non-fuzz crate; do not weaken it during the edition migration.
- Floating dependency ranges in `Cargo.toml` (`clap = "4"`, `serde = "1"`, `serde_json = "1"`, `proptest = "1"`) stay unchanged; only `Cargo.lock` is refreshed.

## File Structure

- `mise.toml` (modify) — base `[tools]`: rust with components; add `fmt-check` task.
- `mise.dev.toml` (create) — env `dev` `[tools]`: the four pinned cargo dev-tools.
- `Cargo.toml` (modify) — `edition` and `rust-version` in `[workspace.package]`.
- `Cargo.lock` (modify) — regenerated by `cargo update`.
- `.github/workflows/ci.yml` (modify) — drop rustup-clippy; add fmt-check step; add non-blocking `audit` job + weekly schedule.
- `AGENTS.md` (modify) — install flow via `MISE_ENV=dev`; remove cargo-deny caveat; version/label updates.
- `docs/ARCHITECTURE.md` (check only) — confirm no stale 1.83/edition references.

### Task 6: Update CI workflow

Remove the rustup-clippy workaround, add the format check, and add a non-blocking scheduled supply-chain audit job.

**Files:**
- Modify: `.github/workflows/ci.yml`

**Interfaces:**
- Consumes: `fmt-check` task (Task 1); `mise.dev.toml` dev env + `audit` task (Task 2); resolved audit (Task 3).
- Produces: a CI file with a `build-test-lint` job (no rustup step) and a non-blocking `audit` job.

- [ ] **Step 1: Replace the workflow with the two-job version**

Overwrite `.github/workflows/ci.yml` with:

```yaml
name: CI

on:
  pull_request:
  push:
    branches: [main]
  schedule:
    - cron: "0 6 * * 1"   # weekly, Monday 06:00 UTC — audit job

jobs:
  build-test-lint:
    name: build + test + lint (rust 1.97, mise-pinned)
    runs-on: ubuntu-latest
    steps:
      - uses: actions/checkout@v4
      - uses: jdx/mise-action@v2
      - name: Format check
        run: mise run fmt-check
      - name: Build
        run: mise run build
      - name: Test
        run: mise run test
      - name: Lint (clippy -D warnings)
        run: mise run lint

  audit:
    name: supply-chain audit (non-blocking)
    runs-on: ubuntu-latest
    continue-on-error: true
    env:
      MISE_ENV: dev
    steps:
      - uses: actions/checkout@v4
      - uses: jdx/mise-action@v2
      - name: Audit (cargo audit + cargo deny)
        run: mise run audit
```

Note: the manual `rustup component add clippy` step is intentionally gone — clippy now ships via the mise pin's `components`. `MISE_ENV: dev` is set at job level so both the mise install and `mise run audit` see the dev tools.

- [ ] **Step 2: Validate the YAML parses**

Run: `python3 -c "import yaml,sys; yaml.safe_load(open('.github/workflows/ci.yml')); print('yaml ok')"`
Expected: `yaml ok`.

- [ ] **Step 3: Sanity-check the referenced tasks exist locally**

Run: `mise tasks 2>&1 | grep -E 'fmt-check|build|test|lint|audit'`
Expected: all five task names appear.

- [ ] **Step 4: Commit**

```bash
git add .github/workflows/ci.yml
git commit -m "ci: drop rustup-clippy workaround, add fmt-check + non-blocking audit job"
```
